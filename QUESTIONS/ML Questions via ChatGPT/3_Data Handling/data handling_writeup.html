How do you handle missing data?

Handling missing data is a crucial step in data preprocessing. The strategy depends on the amount, pattern, and type of missingness. Here are the common techniques:

üîç Step 1: Understand the Missingness

Types of Missing Data:

  1. MCAR (Missing Completely at Random): No pattern.

  2. MAR (Missing at Random): Related to observed data.

  3. MNAR (Missing Not at Random): Related to unobserved data.

Use .isnull().sum() in Pandas or visualizations (like missingno) to inspect.

üß∞ Step 2: Common Handling Techniques

1. Remove Missing Data

  • Drop rows with missing values:

    python
    df.dropna(inplace=True)

  • Drop columns with too many missing values:

    python
    df.drop(columns=['column_name'], inplace=True)

2. Impute Missing Data

Numerical Features:

  • Mean/Median/Mode:

    python
    df['age'].fillna(df['age'].mean(), inplace=True)

  • Interpolation:

    python
    df['temperature'].interpolate(method='linear', inplace=True)

Categorical Features:

  • Mode:

    python
    df['gender'].fillna(df['gender'].mode()[0], inplace=True)

Advanced Imputation:

  • KNN Imputer (from sklearn.impute)

  • Iterative Imputer (like multivariate imputation)

  • Deep learning / ML-based imputation (e.g., autoencoders)

üîÅ Step 3: Add a Missing Indicator

Sometimes helpful:

python
df['feature_missing'] = df['feature'].isnull().astype(int)

üß™ Step 4: Consider Model Type

Some models cannot handle missing values (e.g., linear regression), while others like XGBoost and CatBoost can handle them natively.

‚úÖ Best Practices

  • Analyze why data is missing.

  • If too many values are missing and cannot be recovered, consider dropping the feature.

  • Always validate results after imputation.

What techniques do you use for feature selection?

Feature selection is a crucial step in machine learning that helps reduce overfitting, improve model performance, and make models more interpretable. Here are common feature selection techniques categorized by type:

‚úÖ 1. Filter Methods (Univariate Statistics)

These methods evaluate features independently of the model.

  • Correlation (for continuous variables):

    • Remove highly correlated features (e.g., Pearson correlation)

  • Chi-Square Test (for categorical features):

    • Measures dependence between features and the target

  • ANOVA F-test (for classification problems)

  • Mutual Information:

    • Measures non-linear relationships

python
from sklearn.feature_selection import SelectKBest, f_classif
SelectKBest(score_func=f_classif, k=10).fit_transform(X, y)

‚úÖ 2. Wrapper Methods

These use a machine learning model to evaluate the importance of features.

  • Recursive Feature Elimination (RFE):

    • Recursively removes least important features based on model weights

python
from sklearn.feature_selection import RFE
from sklearn.linear_model import LogisticRegression
rfe = RFE(LogisticRegression(), n_features_to_select=5)
X_rfe = rfe.fit_transform(X, y)

  • Forward/Backward Selection:

    • Add/remove one feature at a time based on model performance

‚úÖ 3. Embedded Methods

These perform feature selection as part of the model training.

  • Lasso Regularization (L1 penalty):

    • Shrinks some feature weights to zero

python
from sklearn.linear_model import Lasso
model = Lasso(alpha=0.01)
model.fit(X, y)

  • Tree-Based Feature Importance (Random Forest, XGBoost, etc.):

    • Automatically rank features by importance

python
from sklearn.ensemble import RandomForestClassifier
importances = RandomForestClassifier().fit(X, y).feature_importances_

‚úÖ 4. Dimensionality Reduction (Optional)

Technically not feature selection, but reduces features to combinations.

  • PCA (Principal Component Analysis)

  • t-SNE / UMAP (for visualization)

üß† Best Practices

  • Use domain knowledge to remove irrelevant features first.

  • Always apply feature selection after train-test split (within a pipeline or cross-validation) to avoid data leakage.

  • Compare models with and without feature selection.

How do you treat categorical variables for ML models?

Treating categorical variables properly is essential for building effective machine learning models. Here's a structured overview of how to handle them:

üîπ 1. Identify Categorical Variables

Categorical variables are usually of type object or category in pandas, and represent:

  • Nominal: No inherent order (e.g., color: red, blue, green)

  • Ordinal: Have a meaningful order (e.g., education level: high school < bachelor < master)

üîπ 2. Encoding Techniques

‚úÖ A. Label Encoding

  • Converts each category to an integer (e.g., red ‚Üí 0, green ‚Üí 1)

  • Suitable for ordinal features

  • ‚ö†Ô∏è Not ideal for nominal features with no order (can mislead models)

python
from sklearn.preprocessing import LabelEncoder
le = LabelEncoder()
df['encoded_col'] = le.fit_transform(df['category_col'])

‚úÖ B. One-Hot Encoding

  • Creates binary columns for each category (e.g., "color_red", "color_blue")

  • Best for nominal features

  • Can increase dimensionality if many unique values

python
pd.get_dummies(df, columns=['category_col'], drop_first=True)

Or using scikit-learn:

python
from sklearn.preprocessing import OneHotEncoder
ohe = OneHotEncoder(drop='first', sparse=False)
encoded = ohe.fit_transform(df[['category_col']])

‚úÖ C. Ordinal Encoding

  • Encode based on known order of categories

  • Specify the order manually

python
from sklearn.preprocessing import OrdinalEncoder
encoder = OrdinalEncoder(categories=[['low', 'medium', 'high']])
df['encoded'] = encoder.fit_transform(df[['education_level']])

‚úÖ D. Target / Mean Encoding

  • Replace category with the mean of the target variable for that category

  • Useful for high cardinality

  • ‚ö†Ô∏è Be careful to avoid data leakage (must be done within cross-validation)

python
df['encoded'] = df.groupby('category_col')['target'].transform('mean')

‚úÖ E. Frequency Encoding

  • Replace each category with its frequency in the data

python
freq = df['category_col'].value_counts()
df['encoded'] = df['category_col'].map(freq)

üîπ 3. Model Compatibility

üß† Best Practices

  • Use one-hot encoding for nominal variables in linear models.

  • Use label/ordinal encoding only when order matters.

  • Consider target/frequency encoding for high-cardinality features.

  • Always encode within cross-validation or a pipeline to prevent leakage.
