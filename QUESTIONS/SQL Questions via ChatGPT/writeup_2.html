What are the different types of joins? Can you explain them with examples?

Great question! Understanding different types of SQL joins is fundamental for working with relational databases.

üîó 1. INNER JOIN

  • Returns only the rows where there's a match in both tables.

sql
SELECT *
    FROM employees e
    INNER JOIN departments d ON e.dept_id = d.dept_id;

‚úîÔ∏è Use when you want only matching records from both tables.

üåê 2. LEFT JOIN (LEFT OUTER JOIN)

  • Returns all rows from the left table, and matching rows from the right. If no match, NULLs are returned from the right.

sql
SELECT *
    FROM employees e
    LEFT JOIN departments d ON e.dept_id = d.dept_id;

‚úîÔ∏è Use when you want all employees, even if they don‚Äôt belong to a department.

üåê 3. RIGHT JOIN (RIGHT OUTER JOIN)

  • Returns all rows from the right table, and matching rows from the left.

sql
SELECT *
    FROM employees e
    RIGHT JOIN departments d ON e.dept_id = d.dept_id;

‚úîÔ∏è Use when you want all departments, even if they have no employees.

üåÄ 4. FULL OUTER JOIN

  • Returns all rows from both tables. Matches where possible, NULLs where not.

sql
SELECT *
    FROM employees e
    FULL OUTER JOIN departments d ON e.dept_id = d.dept_id;

‚úîÔ∏è Use when you want a full picture of both tables, even without perfect matches.

‚ùå 5. CROSS JOIN

  • Returns the Cartesian product ‚Äî every combination of rows from both tables.

sql
SELECT *
    FROM employees e
    CROSS JOIN departments d;

‚úîÔ∏è Use rarely, e.g., when generating combinations or testing.

üß† 6. SELF JOIN

  • A table joined to itself ‚Äî often used to compare rows in the same table.

sql
SELECT a.name AS emp1, b.name AS emp2
    FROM employees a
    JOIN employees b ON a.manager_id = b.emp_id;

‚úîÔ∏è Use for hierarchical structures like org charts or referral relationships.
